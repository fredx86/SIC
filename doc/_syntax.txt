Rule    Brief explanation    Example
digit    /    /
num    /    /
alpha    /    /
word    /    /
alnum    /    /
eol    Newline    /
|    Alternation    alpha | digit
*    Repetition    *alpha
" ... "    Terminal string    "hello"
` ... `    Nocase terminal string    `hEllO`
:    Saving    alpha:my_alpha
[ ... ]    Optional    [ ":" num ]
( ... )    Priority    "a" ( "b" | "c" )
$    Whitespaces    "a" $ "b"
~    Byte    ~127
#    Range    #0-31

Rules that contains a '_' as prefix name are the only one considered for error display
A valid rule name is an identifier. Identifier are defined as ( alpha | '_' ) *( alphanum | '_' )
